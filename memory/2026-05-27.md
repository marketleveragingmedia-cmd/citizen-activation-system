### Technical Lessons Learned

**1. Database Credentials Location:**
- Production DATABASE_URL only exists in Vercel's environment variables dashboard
- NOT stored in local .env files (all have placeholders)
- Neon console is the source of truth for connection strings

**2. Vercel API Route Caching:**
- New API routes can take 60+ seconds to deploy
- Aggressive caching causes 404s on new endpoints
- Direct SQL scripts are more reliable for one-time operations

**3. Build Memory Limits:**
- Complex Prisma queries can cause "KILLED" errors during build
- Simplify queries to direct filters vs recursive hierarchies
- Keep queries focused on single-level relationships

**4. Browser Caching:**
- Users see cached HTML even after deployments
- Always recommend hard refresh (Ctrl+Shift+R) or Incognito mode
- Cache can persist for hours
